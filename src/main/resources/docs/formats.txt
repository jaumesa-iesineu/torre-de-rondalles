TORRE DE RONDALLES - DOCUMENTACIÓ DE FORMATS DE FITXER
=======================================================
Autors: jaume, dani, sergi, kanhai, pere


FITXERS DEL PROJECTE
--------------------
  game.json          -> configuració del joc (mapes, ordre, enemics)
  partida.save       -> partida guardada en binari (a l'arrel del projecte)
  rondalles.db       -> base de dades SQLite (a l'arrel del projecte)
  mapes/planta1.map  -> mapa en text pla
  mapes/planta2.xml  -> mapa en format XML (habitacions)


=============================
1. game.json
=============================

Fitxer de configuració principal del joc. Format JSON, llegit amb Gson.
Classe Java: ConfigGame, MapaConfig, TipusEnemic (paquet joc)
Carregador:  CarregadorGame.carrega("game.json")

Estructura:

{
  "mapaInicial": "<id>",      <- id del mapa on comença el jugador
  "mapaPerdre":  "<id>",      <- id del mapa que es carrega si el jugador perd
  "ordre": ["id1","id2",...], <- ordre de progressió dels nivells
  "mapes": [
    {
      "id":     "<id>",       <- identificador únic del mapa
      "fitxer": "<ruta>"      <- ruta relativa des de resources/
    },
    ...
  ],
  "tipusEnemics": [
    {
      "simbol":  "<char>",    <- caràcter que representa l'enemic al mapa (ex: "M")
      "nom":     "<nom>",     <- nom per mostrar en combat
      "vida":    <int>,       <- punts de vida màxima
      "atac":    <int>,       <- dany per cop (9999 = mata instantàniament)
      "radi":    <int>,       <- radi de detecció del jugador (0 = no detecta)
      "colorR":  <0-255>,     <- component R del color al terminal
      "colorG":  <0-255>,     <- component G del color al terminal
      "colorB":  <0-255>,     <- component B del color al terminal
      "estatica": true|false  <- true = no es mou (trampa)
    },
    ...
  ]
}

Enemics actuals:
  Simbol  Nom              Vida  Atac  Radi  Color R,G,B
  'd','e' DimoniBoiet        10     5     5  180, 40, 10
  'B'     Bubota             20     5     7  180,180,255
  'G'     Gegant             40    10     4  200,120, 50
  'D'     Drac               50    20     8  220, 30, 30
  'M'     NaMariaEnganxa     30  9999     0  180, 50,220  (estatica=true)


=============================
2. partida.save  (binari)
=============================

Fitxer binari que guarda l'estat de la partida en curs.
Classes Java: GestorPartida (desa/carrega), paquet joc
Ruta:        partida.save a l'arrel del projecte

S'usa DataOutputStream (Java) per escriure i DataInputStream per llegir.
Tots els shorts s'escriuen en big-endian (comportament per defecte de Java).
Les cadenes van precedides de la seva longitud en bytes (short), sense null-terminator.

Estructura byte a byte:

  [4 bytes]  Magic: 'S','A','V','E'   <- comprova que el fitxer és vàlid
  [1 byte]   Versió: 1

  --- MAPA ---
  [2 bytes]  longitud id mapa (short)
  [N bytes]  id mapa UTF-8             ex: "planta1"

  --- JUGADOR ---
  [2 bytes]  posX  (short)
  [2 bytes]  posY  (short)
  [2 bytes]  vida actual (short)
  [2 bytes]  vidaMaxima (short)
  [2 bytes]  atac (short)
  [2 bytes]  defensa (short)
  [1 byte]   torns de verí restants
  [1 byte]   torns de foc restants
  [1 byte]   torns de gel restants

  --- INVENTARI ---
  [1 byte]   nombre de slots (sempre 4)
  Per cada slot:
    [1 byte]   0=buit, 1=ple
    Si ple:
      [2 bytes] longitud nom item (short)
      [N bytes] nom item UTF-8      ex: "Poció de vida"
      [2 bytes] quantitat (short)

  --- ENEMICS MORTS ---
  [2 bytes]  nombre d'enemics morts (short)
  Per cada enemic mort:
    [2 bytes]  posX original (short)
    [2 bytes]  posY original (short)

  --- FOG OF WAR ---
  [2 bytes]  amplada del mapa (short)
  [2 bytes]  alcada del mapa (short)
  [A*H bytes] caselles explorades (1 byte cada una: 0=no explorada, 1=explorada)

Com guardar: ESC -> menú pausa -> "Guardar"
Com carregar: ESC -> menú pausa -> "Carregar" (només si existeix partida.save)


=============================
3. rondalles.db  (SQLite)
=============================

Base de dades SQLite que replica les dades del game.json.
Classes Java: PartidaRepository (paquet db)
URL JDBC:    jdbc:sqlite:rondalles.db  (a l'arrel del projecte)

S'inicialitza automàticament en arrencar el joc. Si les taules estan buides,
s'omplen amb els valors per defecte del game.json.

Taules:

  configuracio
  ------------
  clau  TEXT PRIMARY KEY    <- "mapaInicial" o "mapaPerdre"
  valor TEXT NOT NULL       <- id del mapa corresponent

  mapes
  -----
  id     TEXT PRIMARY KEY   <- id únic del mapa (ex: "planta1")
  fitxer TEXT NOT NULL      <- ruta al fitxer .map o .xml (ex: "mapes/planta1.map")
  ordre  INTEGER NOT NULL   <- posició en la progressió (1=primer, 5=darrer)

  tipus_enemics
  -------------
  simbol   TEXT PRIMARY KEY <- caràcter al mapa (ex: "M")
  nom      TEXT NOT NULL    <- nom per mostrar
  vida     INTEGER NOT NULL <- punts de vida màxima
  atac     INTEGER NOT NULL <- dany per cop
  radi     INTEGER NOT NULL <- radi de detecció
  color_r  INTEGER NOT NULL <- component R (0-255)
  color_g  INTEGER NOT NULL <- component G (0-255)
  color_b  INTEGER NOT NULL <- component B (0-255)
  estatica INTEGER NOT NULL <- 0=es mou, 1=estàtica (trampa)

Per consultar amb sqlite3:
  sqlite3 rondalles.db ".tables"
  sqlite3 rondalles.db "SELECT * FROM tipus_enemics;"
  sqlite3 rondalles.db "SELECT * FROM mapes ORDER BY ordre;"


=============================
4. Formats de mapa existents
=============================

  .map   -> text pla, una línia per fila, caracters directes
  .game  -> XML antic amb habitacions connectades per portes
  .xml   -> XML nou amb habitacions i tiles embedded

Carregador: CarregadorMapa.carrega(ruta) detecta el format automàticament.

Caràcters especials als mapes:
  '#'  paret (no travessable)
  '.'  terra (travessable)
  '@'  posició inicial del jugador
  'i'  item al terra (poció)
  'd'  DimoniBoiet
  'e'  DimoniBoiet (alias)
  'B'  Bubota
  'G'  Gegant
  'D'  Drac
  'M'  NaMariaEnganxa (trampa estàtica)
